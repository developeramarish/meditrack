# MediTrack Theming Guide

> **The answer:** A new tenant color palette takes **5 minutes** to apply. Add one object to `color-themes.ts`. No CSS, no component changes, no manual HSL.

---

## For Developers: Adding a New Theme

### Step 1: Pick 5 Brand Colors

Get them from [Coolors.co](https://coolors.co), [Realtime Colors](https://realtimecolors.com), or a tenant/designer.

Map to roles:

| Role | What it is | Pick the... |
|------|-----------|-------------|
| `background` | Page canvas | Darkest (dark theme) or lightest (light) |
| `foreground` | Primary text | Highest contrast against background |
| `primary` | Brand / CTA | Most vivid / saturated |
| `secondary` | Supporting action | Second most vivid |
| `accent` | Highlights / badges | Third color |

### Step 2: Add One Object

```typescript
// design/src/shared/config/color-themes.ts (+ copy to web)
{
  id: "sunset-clinic",
  label: "Sunset Clinic",
  description: "Warm amber medical aesthetic",
  mode: "dark",
  palette: {
    background: "#1a0a00",
    foreground: "#fff4e6",
    primary: "#ff8c00",
    secondary: "#ffd700",
    accent: "#ff6347",
    // Optional — override auto-harmonized semantic colors:
    // success: "#34D399",  // custom emerald
    // warning: "#FBBF24",  // custom gold
    // error: "#EF4444",    // custom red
    // info: "#3B82F6",     // custom blue
  },
  swatches: ["#1a0a00", "#ff8c00", "#ffd700", "#ff6347", "#fff4e6"],
},
```

### Step 3: Done

Theme appears in the ThemeSwitcher automatically. The derivation engine generates **100+ CSS variables** at runtime:

- **33 core vars** — background, card, popover, muted, primary, sidebar, chart, etc.
- **33 brand scale vars** — primary/secondary/accent 50-950 (11 shades each)
- **40 semantic scale vars** — success/warning/error/info 50-900 (10 shades each, harmonized with primary hue)
- **7 healing vars** — healing 50-600 (derived from secondary biased toward teal)

### Perceptual Mapping (no `dark:` overrides needed)

All color scales use **perceptual mapping** — shade numbers have consistent semantic meaning:

| Shade | Meaning | Light mode | Dark mode |
|-------|---------|------------|-----------|
| 50 | Subtle background | Very light tint | Dark muted |
| 200 | Border / separator | Light border | Dark border |
| 500 | Solid / base | Vivid | Vivid (boosted) |
| 700 | Prominent text | Dark text | Bright text |

`bg-success-50 text-success-700` works in **any theme, any mode**. No `dark:` prefix needed.

### Optional: Custom Semantic Colors

By default, the engine harmonizes success/warning/error/info hues toward the primary. To override:

```typescript
palette: {
  // ...5 brand colors...
  success: "#2DD4A8",  // aurora green (Northern Lights theme)
  warning: "#E8A838",  // polar sunrise amber
  error: "#E06B6B",    // soft coral
  info: "#5BC0EB",     // glacial ice blue
},
```

## Architecture

```
┌──────────────────────────────────────┐
│  Component Layer                     │  bg-success-50 text-success-700
│  (90+ .tsx files)                    │  NEVER changes for theming
└──────────┬───────────────────────────┘
           │ resolves via Tailwind + cssScale() to
┌──────────▼───────────────────────────┐
│  CSS Variables Layer                 │  hsl(var(--success-50)), hsl(var(--success-700))
│  (tailwind.config.ts + index.css)    │  Defaults in :root/.dark, overridden at runtime
└──────────┬───────────────────────────┘
           │ values generated by
┌──────────▼───────────────────────────┐
│  Derivation Engine                   │  deriveTheme(palette) → 100+ CSS vars
│  (themeDerivation.ts)                │  Perceptual scales, hue harmonization
└──────────┬───────────────────────────┘
           │ injected by
┌──────────▼───────────────────────────┐
│  Runtime Hook                        │  <style>html.dark { --success-50: ... }</style>
│  (use-color-theme.ts)                │  Cached, localStorage persisted
└──────────────────────────────────────┘
```

## Key Files

| File | Project | Purpose |
|------|---------|---------|
| `shared/config/color-themes.ts` | Both | Theme definitions (palette + semantic overrides) |
| `shared/utils/themeDerivation.ts` | Both | Engine: 5 hex → 100+ HSL vars (scales + harmonization) |
| `shared/utils/badgeStyles.ts` | Web | Centralized badge variant classes |
| `hooks/use-color-theme.ts` | Both | Runtime `<style>` injection + localStorage |
| `hooks/use-theme.ts` | Both | Light/dark/system mode toggle |
| `components/ThemeSwitcher.tsx` | Both | Sidebar popover + Radix ThemeSwitcherPopover |
| `index.css` | Both | `:root` + `.dark` defaults for all scale CSS vars |
| `tailwind.config.ts` | Both | CSS var → Tailwind bridge via `cssScale()` |

## DO / DON'T

### DO
- Add themes by adding one object to `color-themes.ts`
- Use perceptual scale classes: `bg-{color}-50 text-{color}-700`
- Use semantic tokens: `bg-card`, `text-foreground`, `border-border`
- Dual-update: copy changed files to both design/ and web/

### DON'T
- **NEVER** add `dark:bg-{color}-900/30` overrides — perceptual mapping handles it
- **NEVER** use `bg-white`, `text-neutral-*`, or inline hex in components
- **NEVER** manually write CSS for a new theme — the engine generates everything
- **NEVER** modify component files for theming — themes are config-only

## Healthcare Color Guidelines

- **Primary:** Blues (trust, calm) — 85% of healthcare logos use blue
- **Secondary:** Teal/green (healing, growth)
- **Avoid:** Pure black, neon colors, bright red as primary
- **Accessibility:** WCAG AA minimum (4.5:1 text, 3:1 UI)
